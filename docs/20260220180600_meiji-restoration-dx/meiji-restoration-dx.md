# なぜ「令和の変革」は失敗するのか

- 外圧が「見えにくく」なった時代の変革困難
- 黒船は目に見えた ─ クラウドやAIの脅威は「まだ先」に見える
- 成功体験の呪縛：バブル期の「勝ちパターン」が染み付いている
- 合議制の罠：稟議・根回し文化が意思決定を遅らせる
- 「失敗したくない」文化：実験より安全を優先
- → 外圧がなくても「内発的変革」ができるかが鍵

<!--
明治は外圧があったから変われた。現代は外圧が可視化されにくい。でも実際にはAI・クラウドという「第二の黒船」はもう来ている。それを経営層が認識できているか。
-->

# レガシーシステム = 廃藩置県に相当する決断

- 廃藩置県（1871年）：藩主の権限を一夜にして廃止
- 「3ヶ月で決断・実行」 ─ 段階的移行ではなかった
- レガシーシステムの延命 ≒ 旧藩体制の温存
- 「両方を動かし続ける」コストが組織を疲弊させる
- 技術的負債は「藩の借金」のように雪だるま式に増える
- → いつ「廃藩置県」の決断をするか

<!--
廃藩置県は270の藩を廃止し、47都道府県に再編した。これは現代のレガシーシステムからの完全移行と同じ構造。段階移行ではなく、期限を決めた完全移行が必要。
-->

# 「壊す」と「作る」を同時に：廃仏毀釈の教訓

- 明治政府は神仏分離令を発布 ─ 旧来の文化的権威を解体
- 「壊しすぎ」た結果：文化財の損失・民衆の反発
- DXでも「壊す速度 > 作る速度」になると組織が崩壊する
- 廃仏毀釈の教訓：
-   - 既存資産の価値を正確に評価してから壊す
-   - 代替となる仕組みを先に準備する
-   - ステークホルダーへの説明と合意形成を怠らない

<!--
廃仏毀釈は明治維新の「やりすぎ」の例。DXでも既存業務の価値を無視して刷新すると失敗する。テンプル破壊→データ消失、僧侶失業→スキル喪失という対応関係。
-->

# まとめ：「変革の教科書」は150年前にあった

- 明治維新から学ぶDXの5原則
- 
- 1. 外圧を待たず、自ら危機感を醸成する（黒船を自分で呼べ）
- 2. トップが現場に出る（岩倉使節団の精神）
- 3. アイデンティティを保ちながら仕組みを変える（和魂洋才）
- 4. インフラと人材に先行投資する（殖産興業・学制）
- 5. レガシーの廃止期限を決め、二重投資を終わらせる（廃藩置県）
- 
- 150年前に答えは出ている。問われているのは「実行する意志」だ
