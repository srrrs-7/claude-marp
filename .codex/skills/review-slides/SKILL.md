# Review Slides

1. `docs/` の最新 `.md` を読む（またはユーザー指定ファイル）
2. 以下の観点でレビュー:
   - **構成**: intro → body → conclusion の論理的な流れ
   - **密度**: 1スライド最大5箇条書き。超過をフラグ
   - **コード行数**: 8行推奨、12行を超えていないか。コード+箇条書きの組み合わせ制約に違反していないか
   - **コードスタイル**: front-matterに `section pre code { font-size: 0.6em; }` が含まれているか
   - **空白スライド**: 内容のない空白スライドがないか（front-matter直後の不要な `---` 区切りなど）
   - **コード**: 言語タグ正確か、構文妥当か
   - **図解**: SVG/表が読みやすいか。複雑な図（8+ノード）が単独スライド化されているか
   - **ノート**: 箇条書きの単なる繰り返しでないか
   - **Marp**: `marp: true` フロントマター、テーマ、サイズ
   - **言語**: 設定言語と一致しているか
3. 改善を番号付きリストで提案
4. 承認されたらマークダウンを直接編集
